# 大脑-小脑错误恢复闭环 — 实验全记录与诚实结论

任务: LIBERO `put both the alphabet soup and the tomato sauce in the basket`。
故障注入: 第 95~120 步强制开爪(`action[6]=-1`),让正在搬运的物体掉落。
检测: 每个仿真步实时读**物体真值位姿 + 夹爪位姿**判 抓住/放好/掉落(DROP)/卡住(STALL)。
恢复: 检测到失败 → 换计划 + `model.reset()` → 小脑重新推理重抓。

脚本: `scripts/closed_loop.py`(主控)、`qwen_brain.py`(VL大脑)、`compare3.py`/`compare_sidebyside.py`(并排视频)、`latency_sweep.sh`、`plot_rootcause.py`。

---

## 1. 检测是真的吗?
- **每个仿真步都检测**(20Hz,无间隔),基于物体真实位姿,**检测器不知道注入时刻**,是独立从掉落物理现象发现的。
- 局限: 用了仿真**真值物体位姿**(`body_xpos`)= 假设有完美实时位姿估计器。真机要换轻量视觉检测器。
- 两层时序: 便宜监控每步实时跑 + 贵的大脑只在报警时按需调一次(不要求实时)。

## 2. 恢复机制有用吗? (5个初始化)
| init | 无恢复 | 快反射恢复(启发式,~0延迟) | 千问恢复(真4s) |
|---|---|---|---|
| i0 | ❌ | ✅ | ❌ |
| i1 | ❌ | ❌(本就救不回) | ❌ |
| i2 | ✅(无故障) | ✅ | — |
| i3 | ✅(自愈) | ✅(多余) | ? |
| i4 | ✅(自愈) | ✅(多余) | ❌(帮倒忙) |

- **基础策略常能自愈**(扰动结束自己重抓),很多 init 不需要恢复。
- **恢复净胜利(无恢复✗→有恢复✓)只在 i0,且只有快反射行。**
- **i4: 加千问反而失败** — 基础策略本会自愈,但监控 STALL 太激进、打断了它 + 暂停 → 帮倒忙。
- 病根: STALL 检测分不清"真卡死 vs 慢速自愈"。

## 3. 时延曲线 (启发式 + 人为延迟, init0~4)
剔除无故障的 i2,只看触发恢复的: 延迟 0s → 75%; ≥1s → 50%(持平)。
**只有边界 case i0 对时延敏感; i3/i4 任何延迟都成; i1 零延迟也不成。** 时延是次要因素。

## 4. 根因: 为什么暂停几秒就失败? (核心发现)
逐点轨迹对比(`--trace`,见 `rootcause_distance.png`):

| | 物体停的位置 | 手臂起点 | 之后 |
|---|---|---|---|
| REFLEX 成功 | (-0.05,-0.09) | dxy=0.024 | 朝物体收敛 → dxy→0.009 → 抓住✅ |
| DELAY4 失败 | **(-0.048,-0.090) 完全相同** | **dxy=0.025 相同** | 朝反方向走 → dxy→0.2+ → 卡死❌ |

- **物体停在完全相同、够得着的位置**(不是"滚走了")。手臂起点也相同。
- **暂停后策略把手臂往错误方向(篮子)开,没回去重抓那个就在原地的物体。**
- **确定性验证**: d0 跑5次全成功、d4 跑5次全失败,逐字节相同 = 系统确定性、无随机。**所以暂停是因果,不是噪声。**
- MoDE 看的 obs = 2路相机图 + 关节位置 + 夹爪开合(**无速度**)。暂停改变的是"手臂定住后的姿态/夹爪全开/相机画面",单帧反应式策略据此走错。
- 关联用户主线: 计划文本没能压过"视觉上像在搬运"的偏置 → 呼应 plan-fusion / cross-attention 增强计划条件化的方向。

## 5. 总结论 (诚实)
1. 检测 + **快反射**恢复确实能救真正卡死的情况(i0)。
2. **4s VLM 大脑不能进实时恢复闭环**: 要么时延边界失败,要么打断自愈帮倒忙,要么暂停把反应式策略带偏。
3. 正确架构 = **小脑级快反射(实时、即时打断、不暂停)做救火 + VLM 慢决策(诊断/要不要放弃/换序)按需调用**。
4. 暂停"机械臂不动、环境正常"的实现是对的(手臂漂移仅1cm,物体照常滚)。问题不在实现,在"反应式策略 + 暂停"这个组合。

## 6. 改进: 进度感知卡死检测 (`--smart_stall`)
病根 = STALL 把"慢速自愈"误判成卡死、打断它(i4 帮倒忙)。
改进: STALL 触发条件从"110步没事件" → "110步没事件 **且** 机械臂没在接近任何未放好物体"。
机械臂朝某未放好物体下探(EE-物体距离创新低 >3cm)= 在自愈 → 重置卡死计时、不打断。

验证(同样 i4 + 千问 + 立刻打断):
| | STALL | 结果 |
|---|---|---|
| 无 smart_stall | t111 过早打断自愈 | ❌ 失败 recoveries=3 |
| **有 smart_stall** | t111 被抑制 | ✅ **成功** recoveries=1 |

i0(DROP触发,非STALL)不受影响, 仍成功(无回归)。

**但更广验证(启发式+立刻打断, inits 0-9)显示它不是干净win**:
| | 成功 |
|---|---|
| smart_stall OFF | 6/10 (i1,5,6,8 失败) |
| smart_stall ON | 5/10 (额外 i9 失败 = **回归**) |

诚实结论: smart_stall 只在**慢大脑**场景净有益(阻止"过早打断自愈+暂停"的灾难组合); 用**快反射**时其"距离缩小=在自愈"判据太粗, 会把"路过物体"误判成自愈、抑制真卡死(i9回归), 净收益为负。
**想法对(别打断自愈), 实现太糙, 需要更好的进度信号**(如: 手臂是否在下探到抓取高度 + 朝同一物体持续逼近, 而非单纯水平距离缩小)。保留为默认关的实验 flag。

## 7. 最终定论(细延迟扫描, 推翻了"时延有害"的粗结论)
固定 手臂冻死(漂移1mm) + 同一计划 + 无状态单帧策略, 只变暂停时长:
| 暂停 | 步数 | 物体 | 结果 |
|---|---|---|---|
| 0s | 0 | 还在半空 | ✅ 成功 |
| 0.1s | 2 | 已落桌 | ❌ 失败 |
| 0.25/0.5/1/2s | 5~40 | 已落桌 | ❌ 失败 |

**翻转点在最小延迟(2步/0.1秒)= 物体落地那一刻。0.1秒和4秒一样失败 → 与时延时长无关。**

真因(代码核实: reset两条都调; MoDE obs_seq_len=1 无状态单帧; 专家预计算缓存 need_precompute...=False 从未启用, 路由每步重算 → 无残留历史):
**唯一变量是"小脑接手时物体落地没"。** 物体还在半空、就在夹爪下→第一帧锁定"下探去抓"轨迹→成功; 物体已落桌→小脑给乱动作(往篮子开)→失败。
**最可能机制 = OOD**: 训练demo全是干净成功抓取, 从无"物体掉桌上+手臂悬空+夹爪开+任务中途"这种画面, 策略没学过 → 一看到就乱来。reflex成功只因它在"刚脱手还在半空"那一帧接手(像它学过的抓取前一刻)。

**结论修正**: 核心不是"快反射vs慢大脑时延", 而是**小脑对"掉落后物体落桌"OOD状态不会处理**。
解法方向: ①用"物体掉桌→重抓"恢复数据微调策略(最对症); ②专门的落地物体重抓技能; ③"趁半空抓"太脆不现实。

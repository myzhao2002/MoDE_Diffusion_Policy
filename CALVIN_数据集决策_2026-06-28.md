# 换数据集决策 — 用 CALVIN(给 xattn 实验 + 错误恢复实验)

> 日期 2026-06-28。老师要求:用新交叉注意力(视觉 Q / 分步 plan KV)换**复杂数据集**重测,体现两方案差别。本文是调查后的最终决策 + 行动清单。

## 0. 最终结论(2026-06-28 晚更新 — 取代下文早先的 D_D 方案)
**用 CALVIN,数据走「扩量 VyoJ 的 ABCD_D 子集」:从 `VyoJ/calvin-ABCD-D-subsets` 下 6 个训练子集(≈60万帧,6× 现有),合并成一个 training/,保持 ABCD→D 评测设定。** 扩盘到 ~500G。

**为什么不是满量 D_D(早先方案)**:核实下载源后发现 —— D_D 只有 freiburg 官方源,从 autodl 实测 <0.35MB/s(几天下不完);而你已经在用的 ABCD_D 恰好有 **hf-mirror 快镜像(aria2 -x16 实测 ~10MiB/s,每 29GB 子集 ~48 分钟)**,npz 格式直接可用,且**保持你已跑的 ABCD→D 评测与论文可比**、复用现有 validation,零返工。这条路直接治"数据太少",成本最低。

**为什么不下满 24 子集(676G)**:单卡 5090 batch32 喂不饱满量 ABCD,是算力陷阱;6 子集(~60万帧)是单卡能训充分的甜点。需要再加可随时续下(模块化)。

> 下文 §1–§5 为早先以 D_D 为前提的分析,逻辑(为什么 CALVIN、磁盘约束、方法论)仍有效,但**最终数据源以本节为准**。

## 1. 为什么是 CALVIN(不是别的复杂数据集)

| 候选 | 判定 | 理由 |
|---|---|---|
| **CALVIN** | ✅ 选它 | MoDE 原论文 benchmark;仓库已全套接好(`training_calvin.py`/`config_calvin.yaml`/`hulc_data_module`/`rollout_long_horizon.py`);**长程链式评测**(1000 条、每条 5 个连续子任务)正是 xattn 分步 plan 该发光的复杂度;PyBullet 仿真可注入失败 → **错误恢复实验也靠它** |
| LIBERO-90/100 | ✗ | 仍是短程 pick-place,已验证 libero_10 打平(91.62% v2 ≈ 92.5% v1),拉不开差距 |
| RLBench | ✗ | 长程但要 CoppeliaSim + 全新 dataloader/rollout/plan,从零搭,几个月工程 |
| Meta-World | ✗ | 短程、非链式长程,不比 CALVIN 强 |
| RoboCasa/ALFRED/BEHAVIOR | ✗ | 更重的工程,无 MoDE 集成 |
| 真实 OXE 子集 | ✗ | **无闭环仿真 → 测不了成功率、做不了错误恢复 rollout**,直接否掉目标② |

**关键统一论据**:错误恢复实验(目标②)必须有**闭环仿真**来注入失败 + rollout 测恢复成功率。CALVIN(calvin_env/PyBullet)有,而且你已在 LIBERO 仿真上搭过整套闭环恢复 harness,迁过来即可。CALVIN 同时满足两个目标 —— 别的数据集做不到。

## 3. 磁盘约束(决定 split 的硬条件)

## 4. 两条推进轨道(建议都走)

### Track 1 — 立刻用 vyoj 做 xattn A/B(零下载,GPU 一上就跑)
对照实验比的是 **delta**(xattn 开 vs 关,同数据同 seed),子集完全成立,且快。
```
# 前置
cd /root/autodl-tmp/MoDE_Diffusion_Policy
export MUJOCO_GL=egl HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1
export PYTHONPATH=$PWD:$PWD/LIBERO
PY=/root/miniconda3/envs/lerobot/bin/python

# A) baseline(xattn 关)
$PY mode/training_calvin.py \
  root_data_dir=/root/autodl-tmp/CALVIN-datasets/calvin_vyoj \
  plan_file=/root/autodl-tmp/CALVIN-datasets/plans_calvin.jsonl \
  use_extracted_rel_actions=false \
  model.start_from_pretrained=True model.ckpt_path=/root/autodl-tmp/MoDE_Pretrained \
  devices=1 batch_size=32

# B) xattn 开(视觉Q / 分步plan KV)
$PY mode/training_calvin.py \
  root_data_dir=/root/autodl-tmp/CALVIN-datasets/calvin_vyoj \
  plan_file=/root/autodl-tmp/CALVIN-datasets/plans_calvin.jsonl \
  use_extracted_rel_actions=false \
  model.use_resnet_xattn=True model.xattn_visual_query=True \
  model.start_from_pretrained=True model.ckpt_path=/root/autodl-tmp/MoDE_Pretrained \
  devices=1 batch_size=32
```
⚠️ 风险:10万帧偏少,若两臂都欠拟合到接近 0 成功率会看不出差。缓解:warm-start + 报 **SR@1 子任务成功率**(比 SR@5 整链更敏感)。

### Track 2 — 下载标准 task_D_D(论文可比的干净数字)
需先腾盘(见 §3)。下载脚本走 CALVIN 官方 `download_data.sh D` 或镜像。下完结构 = `task_D_D/{training,validation}`,override `root_data_dir=.../task_D_D`,且 D_D 自带 `extracted/` 时可用 `use_extracted_rel_actions=true` 加速。

## 6. 执行清单(最终方案 — 6 子集扩量,turnkey)

**前提:你在 autodl 控制台把数据盘扩到 ~500G**(数据盘 450G→500G;扩盘可能需停机/重启,所以下载放在扩完之后)。

**Step 1 下载(无需 GPU,~5 小时,screen 里跑,断点续传):**
```
cd /root/autodl-tmp/MoDE_Diffusion_Policy
screen -S dlcalvin
bash scripts/dl_calvin_subsets.sh 0 5      # 下 subset_training_000..005
# 解压在 /root/autodl-tmp/CALVIN-datasets/_subsets_raw/,每个下完即删 zip
```

**Step 2 合并成一个 training/(秒级移动,不占额外空间):**
```
PY=/root/miniconda3/envs/lerobot/bin/python
$PY scripts/merge_calvin_subsets.py \
  --stage /root/autodl-tmp/CALVIN-datasets/_subsets_raw \
  --out   /root/autodl-tmp/CALVIN-datasets/calvin_abcd_6sub/training
# validation 复用现有 vyoj:
ln -s /root/autodl-tmp/CALVIN-datasets/calvin_vyoj/validation \
      /root/autodl-tmp/CALVIN-datasets/calvin_abcd_6sub/validation
```

**Step 3 训练 A/B(GPU 上来后,满量数据上同时跑 baseline+xattn):**
```
export MUJOCO_GL=egl HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1
export PYTHONPATH=$PWD:$PWD/LIBERO
ROOT=/root/autodl-tmp/CALVIN-datasets/calvin_abcd_6sub
PLAN=/root/autodl-tmp/CALVIN-datasets/plans_calvin.jsonl
# A) baseline(xattn 关)
$PY mode/training_calvin.py root_data_dir=$ROOT plan_file=$PLAN \
  use_extracted_rel_actions=false \
  model.start_from_pretrained=True model.ckpt_path=/root/autodl-tmp/MoDE_Pretrained \
  devices=1 batch_size=32
# B) xattn 开
$PY mode/training_calvin.py root_data_dir=$ROOT plan_file=$PLAN \
  use_extracted_rel_actions=false \
  model.use_resnet_xattn=True model.xattn_visual_query=True \
  model.start_from_pretrained=True model.ckpt_path=/root/autodl-tmp/MoDE_Pretrained \
  devices=1 batch_size=32
```
可选:下完后跑 `python preprocess/extract_by_key.py` 建 `extracted/` 再开 `use_extracted_rel_actions=true` 加速 IO。
评测想对齐论文 1000 链:改 rollout 的 num_sequences(现 200)。

**待核(下完第一个子集即可验证)**:① 子集 zip 解压的目录层级(merge 的 find_train_dir 已兼容 flat / training/ 两种);② autodl 有无 `unzip`;③ 合并后 lang 标注数 ≈ 6×。
